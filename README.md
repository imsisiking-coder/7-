# 매일매일 보드게임 기획서

## 1. 프로젝트 개요
- 프로젝트명: 매일매일 보드게임
- 한 줄 소개: 아이의 일상 할 일을 보드게임 방식으로 진행해, 자발적인 습관 형성을 돕는 가족형 앱

## 2. 문제 정의
- 현재 문제: 아이들이 정해진 할 일을 제때 하지 않거나 반복적으로 미루는 상황이 자주 발생
- 원인 가설:
	- 할 일이 재미없고 즉각적인 보상이 부족함
	- 부모의 반복적인 지시가 잔소리로 느껴질 수 있음
	- 아이가 성취감을 체감할 구조가 부족함

## 3. 서비스 목표
- 아이가 "해야 해서"가 아니라 "하고 싶어서" 할 일을 수행하도록 유도
- 부모-아이 간 갈등을 줄이고, 긍정적 상호작용을 늘리는 환경 제공
- 게임형 보상 구조로 일상 루틴(정리, 숙제, 준비물 챙기기 등)을 습관화

## 4. 핵심 타깃(페르소나)
- 1차 사용자(관리자): 아이가 할 일을 잘하지 않아 관리에 어려움을 겪는 부모
- 2차 사용자(플레이어): 게임을 좋아하고 경쟁/성취 요소에 반응하는 아동

## 5. 핵심 기능
### 5-1. 보드게임 기반 할 일 진행
- 아이가 할 일을 완료하면 보드판의 말이 전진
- 진행 과정이 시각적으로 보이기 때문에 성취감을 즉시 제공

### 5-2. 이벤트 칸 시스템
- 보드판 중간에 이벤트 칸 배치
- 이벤트 예시: 추가 전진, 한 턴 쉬기, 보너스 점수 획득 등
- 매일의 반복 루틴에 변수를 주어 흥미 유지

## 6. 부가 기능
- 부모 커스텀 맵 생성 기능
	- 부모가 칸 구성, 이벤트 빈도, 보상 규칙 등을 설정
	- 아이 성향(경쟁형/탐험형)에 맞춘 맞춤형 보드판 운영

## 7. 핵심 화면
- 보드게임판 화면: 현재 위치, 남은 칸, 이벤트 칸 상태 확인
- 순위/진행도 화면: 형제/자매 또는 본인 주간 기록 비교, 누적 점수 확인

## 8. 사용자 흐름(예시)
1. 부모가 오늘의 할 일 목록을 등록
2. 아이가 할 일을 완료하고 체크
3. 완료 즉시 말이 전진하고 이벤트 결과 반영
4. 일/주 단위로 순위와 달성률 확인
5. 부모가 필요 시 다음 주 맵 난이도/보상 규칙 조정

## 9. 기대 효과
- 아이의 자기주도적 할 일 수행률 증가
- 잔소리 중심의 관리에서 게임 중심의 동기부여로 전환
- 반복 행동의 습관화로 장기적인 생활 관리 능력 향상

## 10. 성공 지표(KPI) 제안
- 일일 할 일 완료율
- 주간 연속 수행 일수
- 부모 개입 횟수(잔소리/재촉) 감소율
- 앱 재방문율 및 주간 활동 사용자 수

## 11. 개발 기간 및 일정
- 총 기간: 1개월
- 1주차: 요구사항 확정, 핵심 화면 와이어프레임, 기능 우선순위 정리
- 2주차: 보드게임판/할 일 완료/말 전진 등 핵심 기능 구현
- 3주차: 이벤트 칸, 순위 화면, 부모 커스텀 맵 기능 구현
- 4주차: 통합 테스트, UX 보완, 버그 수정, 시범 운영

## 12. 향후 확장 아이디어
- 시즌 맵/테마 스킨(우주, 바다, 정글 등)
- 친구/가족 협동 모드
- 연령별 난이도 추천 템플릿
- 칭찬 리포트 자동 생성(부모용 주간 요약)

## 13. 현재 구현 상태 (MVP)

아래 기능이 웹 앱으로 구현되어 있습니다.

- 보드게임 기반 할 일 진행
	- 할 일 완료 시 담당 플레이어 말 전진
	- 완료 시 기본 점수 획득
- 이벤트 칸 시스템
	- 추가 전진 / 보너스 점수 / 한 턴 쉬기 이벤트 적용
	- 설정한 빈도에 따라 이벤트 칸 자동 배치
- 순위/진행도 화면
	- 플레이어 점수/위치 기반 순위
	- 오늘 완료율, 주간 활동 횟수, 연속 수행 일수, 부모 개입 횟수 확인
- 부모 커스텀 맵 설정
	- 보드 칸 수, 이벤트 빈도, 기본 점수, 플레이 스타일(경쟁형/탐험형) 조정

## 14. 실행 방법

1. 프로젝트 폴더에서 `index.html` 파일을 브라우저로 열기
2. 오늘의 할 일을 등록하고 완료 버튼으로 진행
3. 커스텀 맵 설정 변경 후 저장하여 난이도/보상 규칙 조정

데이터는 브라우저 LocalStorage에 저장됩니다.
